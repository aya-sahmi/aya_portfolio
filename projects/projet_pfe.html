Projet PFE — Application de Merchandising

Backend Laravel • API REST • Gestion produits & synchronisation stocks

Ce projet de fin d’études présente une application de merchandising conçue pour gérer le catalogue produit, synchroniser les stocks entre plusieurs points de vente et fournir des APIs performantes pour des interfaces web et mobiles.

Laravel API REST MySQL Design Scalable
Voir le code Retour au portfolio

Points d'architecture

  • Séparation claire API / Business logic pour faciliter la maintenance
  • Endpoints REST optimisés pour la synchronisation de stock en batch
  • Gestion auth basée sur rôles (admin, magasinier)

Fonctionnalités clés

  • CRUD complet pour produits et catégories avec gestion d'images
  • Synchronisation des stocks entre points de vente via endpoints planifiés
  • Tableau de bord administrateur avec KPIs (stocks faibles, ventes par point)
  • Gestion des utilisateurs et des rôles (permissions)

Détails techniques & bonnes pratiques

  • Utilisation de migrations et seeders pour reproductibilité des environnements
  • Tests unitaires et d’intégration pour les endpoints critiques
  • Optimisation des requêtes SQL et indexation pour la montée en charge
  • Documentation API (ex : Swagger / Postman) pour faciliter l’intégration frontend
